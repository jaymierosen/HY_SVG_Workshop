What is SVG?

You can always reference the doc: MDN Reference

The xlmns attribute is the declaration of the SVG namespace.

An SVG in its own file should be treated as XML.
Web browsers will not render (draw) the SVG image without the namespace.

An SVG file that's placed inside of an HTML file does not need this declaration.

Try: Have a go at deleting the xlmns atrributes and see what happens.

The x and y attributes are the starting coordinates of the SVG

The width and height attributes are designated here as well to set the width and height of the SVG.

It generally corresponds to the values in viewBox attribute

The coordinate system used for the x- and y-positions is similar to many computer graphics and layout programs. The x-axis goes from the left of the page to the right in increasing value, while the y-axis goes from the top of the page to the bottom. This means that the default zero-point, or origin, of the coordinate system is located at the upper left hand corner of the window.

The space is defined in units rather than pixels, so that it can be scaled.

The viewBox attribute defines the position and dimension of the SVG viewport.

The values it takes are min-x, min-y, width, height

Provides a logical structure to the shapes in your graphic

Styles applied to the group will be inherited by the children elements within it.

However geometric attributes (specific depending on the shape, such as circle radius) are not inherited.

These graphical elements are never rendered/drawn directly to the browswer.

To display them, they have to be assigned with an unique id to be referenced later.

SVG's approach to avoiding redundant geometry

It allows you to re-use graphics that you’ve already defined once in your file, to draw the same geometry in multiple places.

  • line is the simplest shape in the SVG lexicon: a straight segment connecting two points
  • x1 and y1are the coordinates for the starting point
  • x2 and y2are the coordinates for the end point

  • Stands for the 'Rectangle' shape.
  • x and yare the starting coordinates of the rectangle.
  • width and height are the width and height of the rectangle.
  • rx and ry is the border radius of the rectangle.

  • The 'Circle' shape.
  • cx (center-x) and cy(center-y) marks the center point of the circle on the SVG coordinate system.
  • r is the radius of the circle.

  • The 'Ellipse' shape.
  • cx (center-x) and cy(center-y) marks the center point of the ellipse on the SVG coordinate system.
  • rx is the horizontal radius of the ellipse.
  • ry is the vertical radius of the ellipse.

  • A basic shape that creates straight lines connecting several points
  • Typically used to create open shapes as the last point doesn't have to be connected to the first point

Reference

  • Like polyline but polygon is a closed shape consisting of a set of connected straight line segments.
  • the shape is automatically closed from the last point back to the first

Reference

  • The drawing toolbox of SVG that can define any shape.
  • You can draw rectangles (with or without rounded corners), circles, ellipses, polylines, and polygons.
  • The d attribute holds the path instruction set, which can be thought of as a separate coding language within SVG. It consists of a sequence of points and single letter instructions, that give the directions for drawing the shape.
  • You can draw shapes that have straight or curved sections (or both), open-ended lines and curves, disconnected regions that act as a single shape, and even holes within the filled areas.
  • It is very rare that you will code a complicated from scratch, this is where SVG visual editors come in handy.

CSS with SVG

Back to Agenda

There are 4 different ways to apply styles on SVG:

  • Presentation Attributes
  • Inline Styles
  • Internal Stylesheets
  • External Stylesheets

CSS in SVG

  • CSS will take priority over a presentation attribute
  • CSS is not required for SVG; it is perfectly possible to define a complete SVG graphic using presentation attributes. But CSS makes it easier.
  • z-index is not yet supported on shape elements within SVG - you must change the order that they appear in the SVG file itself to do that
  • There are a few common CSS properties we use that will not work on SVG elements, let's take a look at a few of them...

The fill property:

  • Sort of like the background property for HTML element.
  • Fills the SVG shapes/path with colour, gradients or patterns.
  • Color Values: can be set with keywords, hex codes #FABDAD, rgb or rgba rgba(250, 189, 173), hsl or hsla hsl(12, 89%, 83%)
  • Gradient and Patterns: set with a url() reference to another element. That other element—the one that describes how to fill in this shape is known as the paint server.
  • fill-opacity: controls the opacity of the fill.

The stroke properties:

  • These are like the border properties for HTML element.
  • stroke: defines the colour (hex code, rgb, hsl)
  • stroke-width: defines the thickness of the stroke
  • stroke-opacity: controls the opacity of the stroke
  • storke-linecap: specifies the shape at the end of a stroke (butt, round, square)
  • stroke-dasharray: controls the pattern of dashes and gaps used to stroke paths (reference)
  • stroke-dashoffset: specifies the distance into the dash pattern to start the dash

transform instead of position

The position property which we often use for animation purposes for HTML elements does not work on SVG elements, we have to use transform instead.

A few things about transform:

  • Transformations can be applied directly to individual shapes or to groups using the SVG transform attribute: transform="scale(4)"
  • The CSS transform property came after the SVG transform attribute, and can still be used on SVG elements: style="transform(scale(4))"
  • ! CSS transform can be implemented inconsistently between browsers

Note on transform-origin property:

  • transform-origin property has inconsistent rendering with SVG
  • when you set transform-origin: center on SVG elements, the expected behaviour is that it will transform at tha center point of the element, which doesn't always work for SVG elements
  • There are several ways to solve this:
  • Consider using a centered coordinate system by modifying the viewBox instead.
  • Use the transform-box property
  • Or if you are using Greensock animation library, it will take care of it for you!

Intro to Greensock

Greensock is a JavaScript library that gives you more control over your animations than just CSS.

  • Workflow is faster and eaiser
  • Sequencing animation is a breeze! No more manually calculating delays!
  • Takes care of those little browser inconsistency bugs! e.g transform-origin!

The video on their homepage explains it well.

The greensock cheatsheet!

Why Greensock?

  • It's still sort of the industry standard
  • Huge support team behind it
  • Robust & Powerful
  • But still very performant!
  • Solves all browser inconsistencies

Intro to Greensock

Greensock Animation Platform (GSAP) core generally consists of:

  • TweenLite - Fast and lightweright tool that consists basic tweening functions
  • TweenMax - Extends Tweenlite but with more capabilities such as staggering, repeat, yoyo etc
  • TimelineLite - Lightweight timeline class for building and managing sequences of Tweens
  • TimelineMax - Extends TimelineLite but with more features
  • There are also lots of GSAP plugins other than the core such as DrawSVG and MorphSVG

Tween: short for in-betweening, the process of generating a smooth transition from one image to another

Greensock: Basic Tween, with TweenLite

What's DrawSVG?

  • A greensock plugin that help us do line animation without manipulating with CSS dash-arrays.
  • You use it by supplying a value to the drawSVGvariable
  • The value you supply describes the end state (or begining of the state if using .from method) of where the stroke line is animating to / from.
  • The value can be in absolute lengths or percentages
  • If 2 values are supplied like so drawSVG: 20% 80%, it means the stroke line will cover 20% from the beginning to 20% from the end of the entire stroke length
  • If only 1 value is supplied like drawSVG: 50%, it defaults as drawSVG: 0 50%, meaning the stroke will cover from starting point to 50% of the stroke length
